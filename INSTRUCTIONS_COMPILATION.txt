ÉTAPE 4 : Si un fichier manque dans l'Explorateur
──────────────────────────────────────────────────
Si vous ne voyez pas un fichier dans l'Explorateur de solutions :

1. Clic droit sur le dossier parent (ex: "DAL")
2. Cliquez sur "Ajouter" > "Élément existant..."
3. Naviguez vers le fichier manquant
4. Sélectionnez-le et cliquez sur "Ajouter"

ÉTAPE 5 : Reconstruire la solution
────────────────────────────────────
1. Menu : Générer (ou Build)
2. Cliquez sur "Régénérer la solution" (ou "Rebuild Solution")
3. REGARDEZ LA FENÊTRE "SORTIE" EN BAS
   - Si vous voyez "Réussite" ou "Succeeded" → C'EST BON ! ✓
   - Si vous voyez des erreurs → Notez-les et continuez ci-dessous

ÉTAPE 6 : Exécuter l'application
─────────────────────────────────
1. Appuyez sur F5
2. Ou cliquez sur le bouton vert "Démarrer" en haut

ERREURS COURANTES ET SOLUTIONS
═══════════════════════════════

❌ Erreur : "Fichier source introuvable"
   → Le fichier existe mais n'est pas dans le projet
   → Solution : Ajoutez-le via "Ajouter > Élément existant..."

❌ Erreur : "Namespace introuvable"
   → Vérifiez que les namespaces sont corrects dans les fichiers

❌ Erreur : "Type introuvable"
   → Vérifiez que System.Configuration est référencé
   → Dans l'Explorateur de solutions, développez "Références"
   → Si System.Configuration manque, ajoutez-le

❌ Erreur : "Cannot find type or namespace"
   → Vérifiez que tous les fichiers sont sauvegardés
   → Vérifiez que les using sont corrects
